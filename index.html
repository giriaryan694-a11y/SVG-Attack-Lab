What This Demonstrates
[ ATTACK EXPLANATION ]

SVG phishing abuses the <foreignObject> tag to embed a complete HTML login form inside an SVG file.

Why it's dangerous:
• Email scanners inspect link destinations, not SVG internals
• A .svg attachment bypasses many content filters
• Looks like a legitimate image attachment
• Form submits silently to attacker server
• No redirect — victim stays on the "page"

Real-world chain:
Email → SVG attachment → User opens → Sees login form → Submits creds → Attacker captures

[ MITIGATIONS ]
• Block SVG attachments in email gateways
• Scan SVG content for <foreignObject>
• User awareness training
• MFA so stolen passwords alone are useless
• Browser isolation / sandboxed SVG rendering

Creds captured by YOUR local python server only.

Note: python3 -m http.server handles GET natively — no extra setup needed. Credentials appear URL-encoded in the request path shown in your terminal.
